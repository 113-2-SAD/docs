## 主要內容

- **[資料庫選型](./week3/database-selection)**：探索不同的資料庫類型 (SQL vs NoSQL)、它們的使用案例，以及為您的專案選擇正確資料庫的標準。
- **[進階資料庫概念](./week3/advanced-database-concepts)**：深入研究分片 (sharding)、複製 (replication)、快取 (caching) 和索引 (indexing) 等技術，以優化資料庫效能和可靠性。
- **[案例研究：Discord 資料庫演進](./week3/discord-case-study)**：從 Discord 擴展其資料庫基礎設施以應對大規模增長的歷程中學習。
- **[RESTful API 基礎](./week3/restful-api)**：理解 REST 的原則、HTTP 方法，以及設計簡潔有效 RESTful API 的最佳實踐。
- **[OpenAPI (Swagger)](./week3/openapi)**：學習如何使用 OpenAPI 來定義、記錄和測試 API，遵循設計優先的方法。

## 學習目標

在本週結束時，您應該能夠：
- 區分各種資料庫類型，並根據專案需求選擇合適的資料庫。
- 理解並應用進階資料庫概念，如分片、複製和快取。
- 透過真實世界的案例研究，理解擴展資料庫系統所涉及的挑戰和策略。
- 遵循最佳實踐設計 RESTful API。
- 利用 OpenAPI (Swagger) 創建全面的 API 文件。
